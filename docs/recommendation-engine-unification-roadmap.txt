Keyboard Recommender — 추천 엔진 단일화 로드맵
==============================================

작성 목적
---------
사용자가 어떤 진입 경로를 선택하더라도 happy path에서는 동일한 FULL 추천 엔진을
경험하도록 정리한다. «빠른 추천»(mode=quick) UI 제거와 엔진 통합이 핵심이며,
스타일 프리셋은 설문 입력 보조만 담당한다.

기준일: 2026-07-09
관련: docs/PROJECT_CONTEXT.md §4.1 · frontend survey-wizard · recommendation_service.py


한 줄 목표
----------
«빠른 추천»을 없애는 것이 목적이 아니라 «추천 엔진을 하나로 통합»하는 것이 목적.
사용자는 어떤 진입 경로를 선택하더라도 결국 동일한 고품질 FULL 엔진을 경험하고,
프리셋은 단지 설문을 더 빠르게 시작하기 위한 입력 보조 역할만 한다.


핵심 원칙 (유지)
----------------
- Happy path: mode=full 단일 경로
- Resilience fallback: full compute 실패 시에만 degraded 경로 (사용자 선택 아님)
- 설문 5축 유지 (trait 엔진과 1:1 — 문항 수 축소 금지)
- responseContractRev: 7 유지 (Phase 1~2; runMode breaking change 시에만 rev 8 검토)
- 스타일 프리셋(ONBOARDING_STYLES) 유지 — chooseStyle() → phase: questions → full submit
- Backend / Frontend / 추천 엔진 아키텍처 전면 재설계 없음


결정 요약 (합의)
----------------
기능                              | 판단
----------------------------------|------------------------------------------
UI Quick Recommendation (quick)     | 제거 (Phase 1)
stable_fast_v1 사용자 경로          | 제거; resilience fallback만 유지 (Phase 2)
스타일 프리셋                     | 유지
프리셋 → FULL 엔진                | 이미 구현 — 깨지 않게 유지
설문 30~60초                      | 목표 OK — 실측 후 UX 개선 (Phase 3)
엔진 단일화                       | happy path 동의; fallback은 예외로 문서화


Phase 개요
----------
Phase 0  베이스라인          제거 전 상태·지표 고정              ~0.5일
Phase 1  Quick UI 제거        사용자 경로에서 mode=quick 차단     1~2일
Phase 2  엔진·계약 정리       fallback 명명·API·테스트 정리       1~2일
Phase 3  설문 UX·실측         30~60초 목표 (실측 기반)            2~4일
Phase 4  문서·운영            PROJECT_CONTEXT · runbook 반영      ~0.5일

의존: Phase 1 → Phase 2 권장. Phase 1과 Phase 3은 병렬 가능.
최소 배포 단위: Phase 1만으로도 제품 방향성 달성.

3.1 실측 (먼저)
---------------
이벤트                                      | 측정
--------------------------------------------|----------------------------------

onboarding.style_selected                   | 프리셋 사용률

목표: P50 설문 완료 ≤ 60초 (프리셋 사용 시 ≤ 45초)


3.2 UX 개선 (우선순위)
----------------------
우선순위 | 개선                                              | 효과
---------|---------------------------------------------------|-------------
P0       | 진입 화면 단순화 (Phase 1 완료 시 상당 부분 달성) | 인지 부하 ↓
P1       | 프리셋 후 pre-filled 단계 스킵                    | 탭 수 ↓
         | seedAnswers로 채운 step «이미 선택됨» 요약 통과   |
P2       | 모바일 option description 접기/툴팁화             | 읽기 시간 ↓
P3       | 진행률 3/5 명확화 + «약 1분» 예상                 | 이탈 ↓
P4       | NL 입력을 설문 후 optional 단계 이동 검토         | 진입 복잡도 ↓

하지 않을 것:
  - 5축 → 3축 축소
  - 프리셋만으로 compute 호출 (설문 생략)
