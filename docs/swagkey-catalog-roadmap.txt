B. 선택적 확장 (키캡·선호 UX)
----------------------------
키캡은 이미 curated 12 + 엔진 6축 + GET /keycaps + /catalog 키캡 탭까지 들어갔다.
남는 것은 **풀을 키우고 / 설문·NL이 키캡을 직접 말하게** 하는 보강이다.

[ ] B1. 키캡 seed 확대
    · 현재: full catalog keycap 65건 중 Base/완결 키트 12건만 seed
    · 목표: trait·spec·shop_view URL이 있는 후보만 추가로 curated 병합
    · 원칙: URL/trait 없이 대량 추가 금지 (핵심 원칙 유지)
    · 산출: seed keycaps[] 증가 · GET /keycaps · regression golden 재생성

[ ] B2. 설문 / NL 키캡 축
    · 현재: 키캡은 엔진·호환 규칙·소리 성향 매칭 위주 (설문 전용 질문·NL 용어 미구현)
    · 목표: 설문에 키캡 프로필/재질 등 선호 질문 추가 · NL에 키캡 용어 → trait delta
    · 관련: ⑭·⑫에서 남겨 둔 설문/NL 후속 항목을 여기로 승격
    · (검토) kit/barebone/complete 설문 질문도 같은 “선호 입력 확장” 계열

[ ] B3. (선택) spec scrape 재실행으로 metadata 정밀도
    · run_all_swagkey_pipeline.cmd — seed sourceUrl 기준 HTML → enrich
    · ⑪에서 남겨 둔 선택 항목 · 추천 풀 품질 보강용

C. 운영 자동화 (ops)
--------------------
지금: fixture 주간 recheck · alert JSON/artifact · live 크롤은 운영자 수동.
남는 것: **실제 크롤을 스케줄로**, **알림을 채널로** 보내는 자동화.

[ ] C1. live crawl 완전 자동화
    · 현재: swagkey-inventory-recheck.yml = fixture diff → alert artifact
             live는 run_swagkey_inventory_recheck.py --mode live (수동)
    · 목표: Actions secret + schedule로 크롤 → 정제 → diff → alert까지 끊김 없이
    · 주의: Swagkey 사이트 부하·차단 · secret에 쿠키/헤더 필요 시 문서화
    · 산출: workflow live job · docs/swagkey-inventory-recheck.md 갱신

[ ] C2. catalog alert webhook
    · 현재: catalog_change_alert.{json,txt} + CI artifact
    · 목표: OPERATIONAL_ALERT_WEBHOOK_URL → Slack/Discord/이메일 등 즉시 통지
    · 트리거: seed_only(품절 후보) · new_in_crawl(신규) · name_changed
    · 산출: alert publisher · env 문서 · staging dry-run
