# UI/UX Improvement Backlog

## 1. Executive Summary

- 현재 UX 수준: 핵심 흐름은 전반적으로 동작하며, 홈 → 설문 → 결과 → 저장/마이페이지까지 큰 뼈대는 갖춰져 있다. 다만 저장 신뢰성 표현, 날짜/시간 표기, 결과/마이페이지의 상태 전달에서 출시 전 보완이 필요한 항목이 남아 있다.
- 가장 큰 강점: 첫 진입에서 "약 1분 · 무료 · 로그인 없이 시작" 메시지와 설문 진입 CTA가 명확하고, 설문 진행률과 결과 요약 구조도 기본 이해가 가능하다.
- 가장 큰 위험: 저장본/계정 영역에 미래 시각이 표시되는 날짜 오류가 실제로 재현되며, 저장 실패 시 원시 오류 문구가 그대로 노출돼 신뢰를 떨어뜨린다.
- 출시를 막는 문제: `2026-07-21` 기준 미래 시각(`2026. 7. 22.`) 노출, 저장 실패 시 복구 불가에 가까운 피드백, 저장 완료 후에도 상태가 불명확한 CTA.
- 출시 후 개선 가능한 문제: 홈의 결과 예고 부족, 설문 액션 위계, 결과 페이지 CTA 중복, 카탈로그/모바일 메뉴의 정보 밀도, 저장/로컬 저장 관련 카피 일관성.

문서와 실제 구현 차이:

- `DESIGN.md`, `DESIGN_SYSTEM.md` 기준 현재의 다크 퍼플 톤은 런치 기준안에 가깝다. 이번 백로그에는 전면 리디자인을 넣지 않았다.
- `docs/home-ia-locked.md`는 홈을 랜딩 중심으로 잠가 두었지만, 실제 로그인 상태 홈에는 "이어서 보기/최근 저장" 스니펫이 일부 노출된다. 이는 즉시 수정 대상이라기보다 정책/일관성 판단 항목이다.
- Launch readiness 문서에서 이미 제외되었거나 잠긴 비교 기능, 대시보드형 홈 회귀, 알고리즘/DB 변경은 범위에서 제외했다.

## 2. 화면별 평가

### 홈

- 현재 상태: 첫 화면 3초 내에 서비스 목적은 이해 가능하다. `약 1분 · 무료 · 로그인 없이 시작`과 설문 CTA도 명확하다.
- 잘된 점: Primary/Secondary CTA 위계가 비교적 분명하고, 설문 시작 목적이 흐려지지 않는다.
- 실제 문제: 게스트 기준으로 결과 예시나 취향 프로필 프리뷰가 없어 "무엇을 얻게 되는가"가 설명보다 약하다. 하단 "다루는 부품"과 설명 섹션은 밀도가 높아 설문 전환 집중도를 조금 떨어뜨린다.
- 개선 가능성: 설명을 줄이기보다 결과 경험 예고를 더 전면에 두는 방향이 효과가 크다.
- 우선순위: `P2`

### 설문 시작 / 빠른 성향 선택

- 현재 상태: 세 카드의 의미는 대체로 이해 가능하고, 선택 시 실제로 중간 문항이 자동 채워진다.
- 잘된 점: 빠른 선택이 장식이 아니라 실제 시간을 줄여 준다는 점이 브라우저에서 확인됐다.
- 실제 문제: 카드 진입 후 설문 하단 액션이 `이전 / 스타일 선택으로 / 처음부터 다시 / 다음`으로 한 번에 노출돼 위계가 복잡하다.
- 개선 가능성: 진입 초반 이탈 감소와 모바일 오탭 방지 효과가 있다.
- 우선순위: `P2`

### 설문 진행

- 현재 상태: 진행률과 문항 수는 정합적이고, 자동 입력 항목도 "스타일에서 이미 선택된 항목"으로 표시된다.
- 잘된 점: 선택 상태와 진행률은 명확한 편이다.
- 실제 문제: `추가 취향 (선택)`이 모든 단계에서 반복 노출돼 핵심 문항 집중도를 떨어뜨린다. 모바일에서는 하단 액션이 많아 시선 분산이 있다.
- 개선 가능성: 주행감 개선 효과가 크다.
- 우선순위: `P2`

### 결과 생성 로딩

- 현재 상태: "추천 조합을 준비하는 중", "잠시만 기다려 주세요", "설문 응답을 반영하는 중…" 정도의 기본 로딩 문구는 있다.
- 잘된 점: 과도한 애니메이션은 보이지 않았다.
- 실제 문제: 왜 기다려야 하는지, 곧 어떤 결과가 나오는지, 실패 시 어떻게 복구되는지에 대한 안내는 부족하다.
- 개선 가능성: 체감 대기 시간을 줄이는 카피 개선 여지가 크다.
- 우선순위: `P2`

### 결과 페이지

- 현재 상태: 추천 제목, 취향 요약, 일치도, 저장/구매 액션이 모두 한 화면 안에 들어온다.
- 잘된 점: 빈 상태는 명확하게 구현돼 있고, 저장 중/성공/실패 상태 영역 자체는 존재한다.
- 실제 문제: 저장 CTA가 상단과 중간에 중복돼 핵심 행동이 분산된다. 저장 성공 메시지가 떠도 버튼 라벨은 다시 `이 빌드 저장`으로 보여 상태가 확정되지 않은 듯 보인다. 오프라인 저장 실패 시 `Failed to fetch`가 그대로 노출된다. 첫 화면 정보 밀도도 높아 요약과 다음 행동의 우선순위가 흐려진다.
- 개선 가능성: 저장 신뢰와 결과 이해도를 동시에 올릴 수 있다.
- 우선순위: `P1` ~ `P2`

### 로그인 / 회원가입

- 현재 상태: `/results`와 `/mypage`에서 들어왔을 때 로그인 이유 문구가 각각 다르게 연결된다.
- 잘된 점: "로그인하면 저장된 추천 결과를 확인할 수 있습니다", "계정으로 로그인하고 저장한 빌드를 이어가세요"처럼 맥락 연결은 잘 되어 있다.
- 실제 문제: 로그인 폼에서 브라우저 `autocomplete` 관련 경고가 확인됐다. 사용성에 즉시 치명적이지는 않지만 자동완성 품질 저하 위험이 있다.
- 개선 가능성: 작은 수정으로 체감 입력 편의가 좋아질 수 있다.
- 우선순위: `P3`

- 미검증: 회원가입 이메일 인증 완료/실패, 비밀번호 재설정 완료 플로우, 인증 후 복귀의 세부 예외 케이스.

### 마이페이지 개요 / 저장한 빌드 / 계정

- 현재 상태: 게스트는 인증 페이지로 유도되고, 로그인 사용자는 개요/저장한 빌드/계정 탭을 볼 수 있다.
- 잘된 점: 개요에서 최근 활동, 저장 수, 최근 로컬 결과 진입점이 정리돼 있다.
- 실제 문제: 저장한 빌드와 계정에서 미래 날짜가 표시된다. `2026-07-21` 현재 기준 저장일과 최근 로그인 시각이 `2026. 7. 22.`로 보여 심각한 시간대/표시 오류다. 저장 목록은 이름이 매우 비슷해 식별성이 낮고, 일부 저장본의 `추천 결과 다시 보기`가 비활성화되어 있는데 이유 설명은 있으나 재활용 가치가 약하다.
- 개선 가능성: 신뢰성과 재방문성에 직접 영향이 있다.
- 우선순위: `P0` ~ `P2`

- 미검증: 닉네임 변경, 비밀번호 변경, 계정 삭제, 아바타 업로드의 실제 성공/실패 플로우.

### 카탈로그

- 현재 상태: 카탈로그와 설문 역할 구분은 문구상 명확하다. 검색과 필터도 동작 구조가 보인다.
- 잘된 점: 로드 실패/빈 상태 카피가 코드상 별도 정리돼 있고, 설문으로 돌아가는 CTA도 있다.
- 실제 문제: 상단 안내 문구와 보조 설명 밀도가 높아 본문 탐색 전 인지 부하가 있다.
- 개선 가능성: 탐색 시작 속도를 높일 수 있다.
- 우선순위: `P2`

- 미검증: 다수 페이지 이동 시 이미지 누락 재현, 모바일 필터 상호작용의 장시간 사용성.

### 모바일 헤더 / 공통 UI

- 현재 상태: 390px, 360px에서 메뉴 자체는 열리고 핵심 링크도 접근 가능하다.
- 잘된 점: 최소 기능 접근성은 확보되어 있다.
- 실제 문제: 모바일 메뉴 안에 검색, 테마, 내비게이션, 로그아웃이 한 번에 들어와 밀도가 높다. 저장/로컬 저장/최근 결과 등 용어도 화면마다 조금씩 달라 학습 비용이 생긴다.
- 개선 가능성: 모바일 핵심 흐름과 공통 카피 일관성이 좋아질 수 있다.
- 우선순위: `P2` ~ `P3`

- 미검증: 포커스 링 키보드 탐색 전수 검증, 모든 터치 영역의 실제 탭 정확도 측정, 전역 에러 페이지 강제 재현.

## 4. 구현 Pass 제안

### Pass 1: 상태·시간·저장 신뢰성

- `SYSTEM-01` 날짜·시간대 표기 오류 수정
- `RESULT-01` 저장 실패 메시지 한국어화 및 복구 경로 제공
- `RESULT-02` 저장 성공 상태를 CTA에 명확히 고정

### Pass 2: 홈·설문 전환율

- `HOME-01` 결과 경험 예고 추가
- `HOME-02` 홈 설명 섹션 밀도 조정
- `SURVEY-01` 설문 하단 액션 위계 단순화
- `SURVEY-02` 추가 취향 입력 노출 위치 재조정
- `LOADING-01` 로딩 카피와 실패 복구 안내 보강

### Pass 3: 결과 페이지

- `RESULT-03` 저장 CTA 중복 정리
- `RESULT-04` 결과 첫 화면 정보 밀도 재정렬

### Pass 4: 로그인·마이페이지

- `AUTH-01` 로그인 폼 자동완성 품질 개선
- `MYPAGE-01` 저장 목록 식별성 강화
- `MYPAGE-02` 저장본 재활용 행동 안내 강화

### Pass 5: 카탈로그·모바일

- `CATALOG-01` 카탈로그 상단 안내 압축
- `MOBILE-01` 모바일 메뉴 재구성

### Pass 6: 공통 UI·카피

- `SYSTEM-02` 저장/로컬 저장/최근 결과 용어 통일

## 5. 하지 말아야 할 개선

- 실제 문제로 재현되지 않은 항목을 체크리스트 문장만으로 확정해 바로 수정하지 말 것
- 현재 런치 기준으로 잠겨 있는 다크 퍼플 시각 언어를 전면 리디자인하지 말 것
- `docs/home-ia-locked.md`를 무시하고 홈을 대시보드형 구조로 되돌리지 말 것
- Launch readiness에서 제외된 비교 기능, 추천 알고리즘, DB 구조 변경을 이번 UI/UX 패스에 포함하지 말 것
- 회원가입 인증 완료/실패, 전역 에러 페이지 등 이번 조사에서 직접 검증하지 못한 항목은 `미검증` 상태로 유지할 것
- 정책 결정이 필요한 항목, 예를 들어 로그인 사용자 홈의 개인화 스니펫 유지 여부는 즉시 수정이 아니라 제품 판단 과제로 분리할 것
- 저장 구조 자체를 바꾸는 큰 개편보다, 먼저 저장 신뢰성 메시지와 시간 표기 오류부터 해결할 것
